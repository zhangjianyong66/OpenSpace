## 二、分类总结

### 2.1 Shell脚本与路径处理 (2条)

#### 教训：Shell路径展开陷阱
- **文件**: LEARN-2026-03-30-121503.md
- **问题**: 在双引号字符串中使用 `~` 表示home目录时，`~` 不会被展开
- **影响**: 创建了以 `~` 为名的字面量目录，导致路径错误
- **正确做法**: 使用 `${HOME}` 代替 `~`，更加明确可靠

```bash
# 错误
LEARNINGS_DIR="~/.openclaw/workspace/.learnings"

# 正确
LEARNINGS_DIR="${HOME}/.openclaw/workspace/.learnings"
```

**建议**: 统一使用 `${HOME}`，在代码审查中注意路径问题。

### 2.3 API开发与限流处理 (1条)

#### 教训：飞书API频率限制处理
- **文件**: LEARN-2026-03-29-140015.md
- **问题**: 批量获取飞书群成员时遇到 429 错误
- **解决方案**: 实现指数退避重试机制

```javascript
async function feishuApiWithRetry(apiCall, maxRetries = 5) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await apiCall();
    } catch (err) {
      if (err.code === 99991400) {
        await sleep(1000 * Math.pow(2, i));
        continue;
      }
      throw err;
    }
  }
}
```

**建议**: 所有飞书API调用都应使用重试包装，记录重试次数并设置告警阈值。

- 模糊匹配时主动询问而非直接拒绝
